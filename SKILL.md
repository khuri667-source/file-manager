# 文件管家 File Manager

## 职责

管理 `E:\ClaudeJobs` 目录，保持整洁、可查找。**只建议，不擅自动手**。

## 触发方式

### 手动触发
用户说出以下任意关键词时激活：
- "清理文件" / "整理文件" / "文件管家"
- "ClaudeJobs 太乱了" / "帮我看看文件"
- 任务完成后说"收拾一下"

### 半自动触发（/remember 联动）
当用户使用 `/remember` 保存偏好后，在保存完毕时主动问一句：
> 最近的任务有需要清理的临时文件吗？要我看一下 `E:\ClaudeJobs` 吗？

用户点头就进入扫描流程。

## 工作流程

### 第一步：扫描

对 `E:\ClaudeJobs` 做三层扫描，输出一份结构化报告：

#### 层 1：临时/垃圾文件
匹配以下特征，统计数量和总大小：
- 通用临时文件：`*.tmp`, `*.temp`, `*.log`, `*.cache`, `*.bak`
- 浏览器/下载残留：`*.crdownload`, `*.part`
- 空文件夹
- 大小为 0 的文件
- 系统文件：`Thumbs.db`, `.DS_Store`, `desktop.ini`

报告格式：
```
🗑️  临时文件：12 个，共 3.2MB
   - output.log (1.2MB)
   - temp_export.tmp (0.8MB)
   ...
```

#### 层 2：同名版本混乱
用文件名相似度判断（不是 hash），找出版本后缀：
- 显式版本号：`v1` / `v2` / `v3` / `final` / `最终版` / `最新` / `old` / `new`
- 日期后缀：`_0714` / `_0715`
- 副本标记：`副本` / `Copy` / `(1)` / `(2)`
- 同一项目名出现多次不同命名

报告格式：
```
📑 同名/多版本文件：
   雁门关PPT美化/
   ├── slides_v1.html  (7月14日)
   ├── slides_v2.html  (7月15日)
   ├── slides_final.html  (7月15日)  ← 最新
   建议：保留 slides_final.html，删除 v1/v2。要执行吗？
```

#### 层 3：无归属文件
根目录下不属于任何项目文件夹的散落文件。

报告格式：
```
📂 无归属散落文件：3 个
   E:\ClaudeJobs\untitled.html
   E:\ClaudeJobs\test.json
   E:\ClaudeJobs\封面图.jpg
```

### 第二步：逐项确认

每一类问题逐条呈现，等用户决定：

| 问题类型 | 默认建议 | 等用户 |
|---|---|---|
| 临时文件 | 删除 | 确认 |
| 多版本 | 保留最新，删旧版 | 用户可能想保留某个中间版本 |
| 无归属文件 | 移动到合适的项目或新建文件夹 | 用户决定归属 |
| 空文件夹 | 删除 | 确认 |

**绝不批量删除**，一类一类、一项一项来。

### 第三步：执行

用户确认一项 → 执行一项 → 报告结果。完成后输出总结：

```
✅ 清理完毕
   删除临时文件：8 个 → 释放 2.1MB
   合并版本文件：保留 3 个最新，删除 5 个旧版
   归类散落文件：2 个移入对应项目
   当前 E:\ClaudeJobs 剩余项目：X 个，总大小 YY MB
```

## 重要规则

- **不自动删任何东西**，每步都等确认
- **不做归档/压缩**，用户会随时查看旧项目
- **文件名判重即可**，不做内容 hash
- **不碰项目文件夹内部结构**，只处理根目录和组织层面
- 操作前先展示文件大小，让用户判断值不值得删
